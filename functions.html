3.1.2 Designing functions

Create functions for tasks that need to be executed frequently.

read_weather_data <- function() {
  # Read all files
  berlin <- readxl::read_excel(here("data/weather", "berlin.xlsx"))
  toronto <- readxl::read_excel(here("data/weather", "toronto.xlsx"))
  tel_aviv <- readxl::read_excel(here("data/weather", "tel_aviv.xlsx"))
  zurich <- readxl::read_excel(here("data/weather", "zurich.xlsx"))

  # Create ensemble dataset
  weather_data <- bind_rows(
    berlin = berlin,
    toronto = toronto,
    tel_aviv = tel_aviv,
    zurich = zurich,
    .id = "city_code"
  )

  # Return it
  weather_data
}

ellipsis_test <- function(...) {

3.3 Scope

When working with functions, it is important to be aware of the scope of objects, i.e.Â what is known in which context?

a <- 5

a is now an object of the global environemnt. A function can access global variables:

f <- function() {
  a
}

f()
## [1] 5

On the other hand, a variable which is defined inside a function, will not be known outside of that function. Respectively, it wonât overwrite the value of global variables.

f <- function() {
  a <- 2
  a
}

f()
## [1] 2
a
## [1] 5

Global variables are a (hidden) part of a functionâs interface.

Best practice: functions should be self-contained, independent of global variables.

3.4 Pure functions and side effects

Functions should do one thing, and do it well.1

Pure function: called for its return value, no side effects:

pure_function <- function(x) {
  x + 1
}

pure_function(1)
## [1] 2

Function with side effect: called for its side effect, returns input (here invisibly; assigning is possible):

side_effect_function <- function(x) {
  file <- tempfile()
  writeLines(format(x), tempfile())
  print(x)
  message(x, " written to ", file)
  invisible(x)
}

side_effect_function(2)
## [1] 2
## 2 written to /tmp/Rtmp3HWC3x/file2b9a1f8afacb
5 %>%
  pure_function() %>%
  side_effect_function() %>%
  pure_function()

## [1] 7

read_weather_data <- function(omit_zurich = FALSE, omit_toronto = FALSE) {
  # Create ensemble dataset from files on disk
  weather_data <- bind_rows(
    berlin = read_weather_file("berlin.xlsx"),

      weather_data %>%

read_weather_data <- function(omit_zurich = FALSE, omit_toronto = FALSE) {
  # Create ensemble dataset from files on disk
  weather_data <- bind_rows(
    berlin = read_weather_file("berlin.xlsx"),
    toronto = read_weather_file("toronto.xlsx"),
    tel_aviv = read_weather_file("tel_aviv.xlsx"),
    zurich = read_weather_file("zurich.xlsx"),
    .id = "city_code"
  )

  # Filter, conditionally, and return
  if (!omit_zurich && !omit_toronto) {
    weather_data
  } else if (omit_zurich && !omit_toronto) {
    weather_data %>%
      filter(city_code != "zurich")
  } else if (!omit_zurich && omit_toronto) {
    weather_data %>%
      filter(city_code != "toronto")
  } else {
    # Filter both
    weather_data %>%
      filter(city_code != "zurich") %>%
      filter(city_code != "toronto")
  }
}

make_read_weather_file <- function(filename) {
  # Avoid odd effects due to lazy evaluation
  force(filename)

  # This function (closure) accesses the filename from the
  # outer function
  function() {
    read_weather_file(filename)
  }
}

loudly <- function(f) {
  force(f)

  function(...) {
    args <- list(...)
    msg <- paste0(length(args), " argument(s)")
    message(msg)

    f(...)
  }
}
